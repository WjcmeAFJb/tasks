# Tasks — Development Notes

## Nix Dev Environment

Enter the dev shell:

```
nix develop
```

This provides: JDK 17, Gradle, Android SDK (platforms 34/35/36), emulator,
system images, and E2E helper scripts. It also auto-updates `local.properties`
with the correct `sdk.dir`.

### Building

```
./gradlew :app:assembleGenericDebug
```

The `generic` flavor builds without Google Play Services dependencies.

### Emulator

```
create-avd          # one-time: creates AVD "test_device"
run-emulator        # starts emulator and blocks until boot completes
```

Or via flake apps outside the dev shell:

```
nix run .#create-avd
nix run .#run-emulator
```

#### SELinux (critical)

On Fedora/RHEL with SELinux **Enforcing**, the Android emulator segfaults
~25 seconds after startup. The crash is in QEMU's memory-mapped rendering
path and affects both the Nix-packaged and Google-distributed emulator
binaries. **Set SELinux to Permissive before launching the emulator:**

```
sudo setenforce 0
```

This is the single most important finding from the initial setup. Without
this, every emulator launch silently crashes with SIGSEGV.

### E2E Helpers (available inside `nix develop`)

| Command | Usage |
|---|---|
| `e2e-screenshot [file]` | `e2e-screenshot home.png` |
| `e2e-tap X Y` | `e2e-tap 276 600` |
| `e2e-text "words"` | `e2e-text "Buy groceries"` |
| `e2e-dump-ui` | Dumps full UI tree (XML) |

These wrap `adb` and work against any running emulator on port 5554.

## Testing

### Unit Tests

```
./gradlew :app:testGenericDebugUnitTest
```

52 unit test files covering recurrence logic, CalDAV/iCalendar parsing,
date utils, and sync logic.

### Instrumented Tests

```
./gradlew :app:connectedGenericDebugAndroidTest
```

67 instrumented test files covering DAOs, services, adapters, and ViewModels.
Requires a running emulator.

### E2E Tests (Maestro)

```
maestro/run-all.sh                    # runs all 8 flows with video recording
maestro test maestro/02_create_task.yaml   # run a single flow
```

8 flows: onboarding, create task, edit task, complete task, navigation
drawer, settings, subtasks, search. Each captures screenshots and video.

Requires: Maestro CLI (`curl -fsSL https://get.maestro.mobile.dev | bash`),
a running emulator, and the app installed.

## Coverage

### Unit Test Coverage

```
./gradlew -Pcoverage :app:testGenericDebugUnitTest :app:jacocoReport
```

Generates HTML + XML reports in `app/build/reports/jacoco/jacocoReport/`.
The `-Pcoverage` flag enables JaCoCo instrumentation.

### E2E Coverage Limitation

`enableAndroidTestCoverage` cannot be used because JaCoCo fails to instrument
the `:icons` module — `OutlinedGoogleMaterial$Icon.<clinit>` exceeds the 64KB
JVM method limit after instrumentation (`MethodTooLargeException`). AGP
transforms ALL dependency jars through JaCoCo with no per-class exclusion.

A debug-only `CoverageBroadcastReceiver` is included for future use once
the icons enum is split:

```
adb shell am broadcast -n org.tasks/.coverage.CoverageBroadcastReceiver \
  -a org.tasks.DUMP_COVERAGE --receiver-include-background
```

## Mutation Testing

```
./gradlew :app:pitestMutationTesting
```

Runs PIT (pitest 1.17.4) against unit tests with the STRONGER mutator set.
Reports (HTML + XML) land in `app/build/reports/pitest/`.

PIT does not natively support Android modules. The Gradle task runs PIT as a
JavaExec with the app's compiled classes and test classpath assembled manually.

Target packages: `org.tasks.repeats`, `org.tasks.caldav`,
`org.tasks.notifications`, `org.tasks.location`, `com.todoroo.astrid.repeats`,
`com.todoroo.astrid.alarms`.

PIT's built-in `+fkotlin` filter handles Kotlin compiler-generated bytecode.
